Preprocessing Pipeline

1
MediaPipe Hand Detection
Each image is processed through Google's MediaPipe Hands solution to detect 21 key landmarks. Confidence threshold: 0.5.
2
Landmark Extraction
Extract (x, y, z) coordinates for all 21 landmarks → 63-dimensional feature vector.
3
Normalization
Coordinates normalized (0-1) by MediaPipe. Additional standardization using training set statistics.
4
Label Encoding
String labels converted to integer indices using LabelEncoder. Mapping saved for inference.
5
Train/Test Split
80-20 stratified split ensures representative class distribution.

Current Limitations & Future Work

Future Work
Multi-Object Tracking
Currently handles single hand only. Multi-hand/multi-person tracking is not implemented yet.
Future Work
Sentence Level Recognition
Model recognizes isolated signs only. Continuous sentence recognition is not supported yet.
Future Work
Additional Classes
More ISL words and phrases to be added in future iterations. Current model supports 43 classes.
Ambiguity
Sign Confusion
May confuse between visually similar signs (e.g., 'M' and 'N'). Accuracy for such pairs is lower.
Tradeoff
Speed-Accuracy Tradeoff
Real-time processing limits model complexity. A deeper model could improve accuracy but reduce FPS.
Known Issue
Lighting Sensitivity
Performance degrades in poor lighting or with harsh shadows on the hand.
